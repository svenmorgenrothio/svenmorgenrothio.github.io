Blog Post

APIs make XSS prevention a frontend job

APIs make XSS prevention a frontend job

This blog post is about the shifting responsibility of XSS prevention from the backend to the frontend. It discusses several aspects of that shift, such as the need for a new approach to output sanitization, which should be unique to each frontend framework that processes the data provided by an API.

Additionally it discusses the need for secure defaults and gives an example of an impactful Cross-Site Scripting vulnerability in a popular mobile application framework.
